# UI-CLI Parity Matrix

Purpose: map each meaningful UI feature to an equivalent CLI workflow (or explicitly classify as UI-only).

## Status Legend

- `implemented`: CLI parity exists and is documented/tested.
- `missing`: no CLI parity exists yet.
- `partial`: some parity exists but gaps remain.
- `ui-only`: intentionally UI-only (must include rationale).

## Severity Legend

- `H`: high parity risk / release blocker.
- `M`: medium parity risk.
- `L`: low parity risk.

## Current Focus

- Branch-close state: parity and unification are complete for the default Design -> Preview -> Run workflow.
- Shared-hook boundary:
  - Room/Attacker/Defender direct authoring parity is implemented,
  - the active UI workflow and CLI run on the same command rails,
  - unnecessary dependency coupling is removed or isolated from the default workflow,
  - deeper IPFS/blockchain/live-runtime-LLM product work continues in follow-on branches.
